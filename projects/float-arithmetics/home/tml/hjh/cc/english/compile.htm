Compilation with different Compilers (overview)

The source files of all programs of Hans-Jürgen Herberts /home/cc - Programs are held in subdirectories /home/cc/.... Download and unpack:

  • C:\HOME\CC - some batch files which help to invoke a compiler 
  • C:\HOME\CC\INCLUDE - All include files. See ../../../../cc/include/ for a list
  • C:\HOME\CC\LQ - All library files. See lq/00idx.htm for documentation. See ../../../../cc/lq/ for a list
  • C:\HOME\CC\UTIL - All utilities (as source). See util/00idx.htm for documentation. See ../../../../cc/util/ for a list
  • C:\HOME\CC\QQ - Various programs which verify a part of the library or some computer hardware. See qq/00idx.htm for documentation. See ../../../../cc/qq/ for a list
  • C:\HOME\CC\HTMUTIL - Work on HTML-Files. See htmutil/00idx.htm for documentation. See ../../../../cc/htmutil/ for a list

All files may be compiled by various compilers for various operating systems:
  • LINUX
    • Gnu Compilers g++ or gpp
  • DOS
    • Borlands Turbo C++ : I use Verion 1.0. And I use only the short memory model.
    • DJ GNU Compiler for DOS. This is a 32 Bit Compiler.
  • WINDOWS95, 3.11, NT
    • Borlands Turbo C++ runs in a DOS-BOX, or in DOS7 mode
    • MSVC10 (16 Bit) The Version 1.0 of Visual C++
    • MDOS40 MSVC40 (32Bit)
    • MDOS50 MSVC50 (32Bit)
  • OSK (OS-9) on a 68020 processor
    See oskgcc : The port of the gnu compiler
    • Gnu 2.7.2 Port and Gnu 2.5.8 Port
  • Atmel AVR with WinAVR http://winavr.sourceforge.net
    See ../../atmel/winavr.htm
It is common to all DOS and Windows-Dos-Prompt compilations that:
  • pre usage of a compiler (from command line), a batch file from C:\home\cc must be invoked to set up the compilers environment and paths. The batch files can be : tc.bat mdos50.bat dj.bat avr.bat.
  • After this, a single xxx.cpp file is compiled (into .Object) by invocation of ccs xxx. Where CCS.BAT is in the search path, in /home/cc/tcbat, /home/cc/mdos50.b, /home/cc/msvc50.b, /home/cc/djbat.
  • For each compiler the .obj files are hold in a different subdirectory ./TC/*.OBJ ./DJ/*.o ./MDOS40/*.OBJ
  • The whole library /home/cc/lq is compiled by invocation of hmake, which interprets hmake.txt
  • All utilities are compiled in subdirectory /home/cc/util by invocation of hmake, which interprets hmake.txt
  • Under LINUX all files are compiled by invocation of the shell script tl (Translate) The makefile is not for Linux, but for OS-9.
  • Under OS-9 all files are compiled by invocation of make which will use the makefile.
  • After compilation all exe files will reside in a subdirectory C:\EXxxxx (for DOS and Windows), in /h0/HOME/CC/CMDS for OS-9; and in ~/home/cc/bin for Linux.
  • All unneccesary files (before saving to a tape, Network or CD) may be deleted by alld cppclean where cppclean is a *.bat file in my search path C:\EXTENDS
