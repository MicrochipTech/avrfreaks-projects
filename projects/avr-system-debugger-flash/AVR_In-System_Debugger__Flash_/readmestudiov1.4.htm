Introduction

Thank-you for downloading this 1st production release version of STUDIO+ 30 Day evaluation software which will allow you to debug your own ATMEL STUDIO (V3.xx) assembler and Imagecraft C programs free for 30 days. Before trying it out with your own programs, please run the supplied ATMEL STUDIO or ImageCraft Bounce demo programs on either a Mega103/64/128 or non-Mega (8515/8535 etc) AVR CPU or any AVR CPU which contains a UART. Once you are familiar with the way the Bounce program works you can try assembling or compiling your own programs using ATMEL STUDIO or an Imagecraft C compiler. If you already own an ATMEL STK200/300/500 starter kit, you can try out the demo programs via the RS232 port of the ATMEL starter kit (see help file for further details). For STK500 kits it is possible to download code via one RS232 and debug via the other RS232 port.

 

AVR EXISTING DESIGNS WITH 10 WAY ISP HEADER

If your existing prototype or production board design contains the standard ATMEL 10 way ISP programming header, we recommend you update your design to the Flash Enhanced 10 way ISP, add the following connections to your ATMEL10 way ISP: CPU Tx (to pin 4) and CPU Rx (to pin 6) on the current ISP these are connected to GND, you can then use the Special Flash RS232 ISP/ISD dongle  to download/debug your prototype/production boards. This is the cheapest way to debug your own boards without having to purchase an ICE. If you use the ATMEL 6 way ISP programming header, we recommend you update your design to the Flash enhanced 8 way ISP: CPU TX (to pin 7) and CPU Rx (to pin 8).

Software Installation & Project Setup

The STUDIO+, AVR 8515/Mega DEMO and ISD files are all zipped up in a file called “STUDIO8515”, un-zip this and then unzip the STUDIO+ v1_3 zip file to a temp directory and click on setup, STUDIO+  will now be installed on your PC. A short-cut is added to your windows START MENU and the STUDIO+ FLASH icon is added to your desktop. If you have installed Imagecraft ICC AVR compiler, you should also download the IMAGEISDLIB.zip files, you cannot use the ICC AVR compiler without the ICCAVRISD library files being present.

 

Make sure the ICC AVR compiler is installed first before trying to install the Imagecraft ISD library.

Copy the “Studio+ ISDs v1.x” file to a  temp directory, unzip to C:\ using the "use folder names" option. After unzipping check that the Imagecraft ISD files are in the following directory: C:\Studio+\ISDs\AVR\ImageCraft\

And the AVR STUDIO ISD is in the following directory:

C:\Studio+\ISDs\AVR\AVR Studio\

To install the Imagecraft ISD library, just click on “MAKELIBS” (DOS batch file) or manually copy across all “*.a” files to the ICC lib directory. Creating other library files for other AVR CPU’s is easy (details supplied if you purchase STUDIO+).

 

Finally to install the Demo programs, unzip “Studio+ v1_x Demos” to c:\ using the “use folder names” option. After unzipping check that the following demo directories have been created:

 

C:\Studio+\Demos\ImageCraft        \AT90S8515_Bounce

                                                            \AT90S8515_Bat-n-Ball

                                                            \ATmega103_Bounce

                                                            \Atmega103_Bat-n-Ball

 

C:\Studio+\Demos\AVR_Studio     \Atmega103_Bounce

                                                            \AT90S8515_Bounce

 

PLEASE NOTE: If you install the demo programs to any other directory than those given above, you will need to re-compile/re-assemble each demo before you load it into STUDIO+.

 

When you run STUDIO+ the first time, it will require you can create a NEW project or OPEN an existing project,  select New to reveal project wizard, now select IMAGECRAFT and open an Imagecraft bounce project from one of the above directories, you should see the bounce Project Files window open, now select PROJECT|SETTINGS and select the appropriate CPU, Package type: for Mega select Mega103/TQFP, for non mega select 8515/DIL, select SFR’s you wish to view by clicking in the tick boxes (we suggest PIND, PORTD, . Check BAUD rate is set correctly (Flash default is 115k BAUD for 3.6864Mhz crystal). Now select VIEW and open up all the following windows, positioning them as you wish:

Microwatch

Stack

MEMORY: SRAM

MEMORY: SFR’s

Chip

Variable Watch

SOURCE: bounce.c

 

In the Project Files window, click on the + sign for bounce.c, click on Global Variables, and right click on memory char ([10][16]) to display three options:

SHOW

ADD TO WATCH

PROPERTIES

 

Click on ADD TO WATCH, you should now see this variable added to the Variable Watch window, if you click on SHOW it will take you direct to the source line where the variable is declared.

Now select SAVE PROJECT and finally exit by selecting FILE| EXIT.

 

Click again on STUDIO+ and now select the project displayed in the FILE list, You should see all the windows now open as they were just before you selected FILE|EXIT. Do not yet play with any of the tool-bar buttons, otherwise you may generate errors or In-System-Programming may not work. You can though drag the Target Control toolbar to anywhere, to speed up debugging.

 

You have now completed the SETUP procedure.

Flash AVR 5in1/Mega8 Module Users

 

Provided with AVR5in1/AVR mega8 is a PROTEL schematic and documentation. If you purchase an AVR Pro Developers pack, we will supply the PROTEL AVR lib to help kick-start your own AVR design if you use Protel software. The kit is provided with links set to the default state of programming via Imagecraft IDE and debugging using STUDIO+. Program the AVRbounce.hex file to an Atmega chip for AVRSM/Mega8 kits or an 8515 chip for AVRDIL kits, using the supplied Windows ISP software. If you are using IMAGECRAFT you can also program the chip direct from the Imagecraft IDE. Now connect up a 9 way RS232 cable between your PC and the AVR RS232 port (TX, RX, GND and DTR etc control connections are required, pins 1 & 9 N/C).

 

RUNNING BOUNCE DEMO

 

Return now to STUDIO+ and click on the com port off/on switch to open the comms port. Now click on the Snapshot button, ATMEL STK users should also manually reset your target. You should now see a “Ball” bouncing around in the SRAM window. Experiment with the other “Target Control” buttons, including “PAUSE”, ANIMATE, C STEP, SINGLE STEP, the RESET TARGET button can only be used with Flash 5in1/AVR modules, this is used to remote reset/run your target. Experiment now, by adding further watch variables, you could also try out the comms self-heal, by un-plugging the comms and then re-connecting, after a few seconds you will see the Target Controls re-activated, you can now continue debugging. Try also the dynamic breakpoint setting feature, just click on any source line, a yellow arrow will appear, you can now run to the new breakpoint (without re-compiling), the bottom right status panel tells you when the target is running. Note that you cannot set/run to a break within the ISD routine, this is the only code you don’t need to debug (guaranteed 100% bug free or you get your money refunded).  Checking for floating CPU inputs is easy, activate snapshot mode and run your finger over the CPU pins, CHIP VIEW will instantly show you, which inputs are floating. Try changing the CPU type, notice STUDIO+ auto detects an-incorrect CPU when it hits a breakpoint. Finally in the watch window, right-click anywhere in the blank area below variables and click on Add New variable, you can now manually set-up a variable to watch (useful for ATMEL STUDIO assembler or other C compiler watches). If you are using a Portable you may want to turn off the Comms Timeout.

 

COMPILING/DEBUGGING YOUR OWN IMAGECRAFT C PROGRAM

After you have run the bounce demo, you are ready to start debugging your own code, we suggest you copy the bounce program across to a new working directory. You will also need to install the AVRISD library files. In the Imagecraft PROJECT | OPTIONS make sure Additional Lib is set correctly (e.g. ISD15AT90S8515 for AT90S8515 CPU). In Imagecraft IDE, create a new project and add the bounce source file (which you may have renamed) to the new Imagecraft Project. Now edit the main Bounce program removing the bounce code and start to add your own code (after the SEI() statement, add at least one hardware break (see our break example). Remember you can add further dynamic breakpoints within STUDIO+. Re-compile and download the hex code to your CPU, in STUDIO+ open  the new Imagecraft project and start to debug in a flash …. after re-compiling, just hit the STUDIO+ Refresh Project button no need to close/re-open STUDIO+.

 

ASSEMBLING/DEBUGGING YOUR OWN ATMEL STUDIO PROGRAM

 

After you have run the AVR STUDIO assembler bounce program, you are ready to start debugging your own assembler programs. You will use ATMEL STUDIO (v3.xx) IDE for editing/assembling and STUDIO+ for debugging. We supply two STUDIO ISD’s which support AT90S8515 and Mega103 chips, we also supply the Universal AVRISD.inc file, each AVR CPU requires a different ISD, it is easy to create ISD’s for other AVR CPU’s. We suggest you copy the main bounce program across to a new working directory and create a new STUDIO project. Add the bounce program (which you may have renamed) to the project plus the correct STUDIO ISD (N.B. ATMEL STUDIO does not support the concept of ISD libraries). Now edit the main program and start to add your code after the SEI instruction, you can remove the other parts of the bounce program, which are not required, add at least one hardware break (see our example). After re-assembling in ATMEL STUDIO, download your code to your CPU, in STUDIO+ open  the new ATMEL STUDIO project and start to debug in a flash …. after re-assembling, just hit the STUDIO+ Refresh Project button no need to close/re-open STUDIO+. TIP: Purchase the Universal ISD programmer from Flash and you can assemble/download via LPT port using ATMEL STUDIO and debug via PC’s RS232 port using STUDIO+.

 

COMPARISON WITH OTHER DEBUGGING TOOLS/METHODS

We have done our own comparison with debugging via simulators, we hope you will agree that debugging this way is much quicker, you know all the internal features of the CPU are working correctly, including timers, PWM, code execution speed etc. STUDIO+ offers a cheaper alternative to ICE (incl JTAG ICE) as it will support ALL CPU’s that contain a UART including all the Mega range (ICE200 users upgrade now to STUDIO+), STUDIO+ also operates much quicker running at 115k baud, it will never need repairing and it is properly supported (bug fixes/updates available from our web site). Multiple license discounts are available for education. If you cannot spare your CPU UART, we will provide a version of the ISD which can operate via two spare I/O pins or the SPI pins.
