libavrutil - LCD driver components

Libavrutil porvides access to the driver protocol of the de facto standard LCD control chip Hitachi HD44780. In order to use this library, one has to connect the AVR to the LCD PCB as follows:

  • PORT B, bit 0: LCD data bit 4 (usually pin 11)
  • PORT B, bit 1: LCD data bit 5 (usually pin 12)
  • PORT B, bit 2: LCD data bit 6 (usually pin 13)
  • PORT B, bit 3: LCD data bit 7 (usually pin 14)
  • PORT B, bit 4: Register Select (usually pin 4)
  • PORT B, bit 5: Clock/Enable (usually pin 6)

Also don't forget to pull the R/~W pin (pin 5) to ground.


void avr_lcd_init(): Must be called before using any other avr_lcd_*() functions.

void avr_lcd_config(uint8_t flags): modifies the appearance settings of the display. Use the AVR_LCD_DISPLAY, AVR_LCD_CURSOR and AVR_LCD_BLINK constants (OR them together) to turn on the display, the cursor and the blinking of the cursor, respectively. Omit them to turn the features off.

void avr_cld_clear(): clears the screen and positions the cursor at row 0 column 0.

void avr_lcd_home(): positions the cursor at row 0 column 0 without clearing the screen.

void avr_lcd_set_cursor_pos(uint8_t addr): Sets the cursor position using the AVR_LCD_LINEn (n = 0 or 1) macros. Add the column number (indexed from 0) to shift the cursor to the right.

uint8_t avr_lcd_get_cursor_pos(): returns the cursor position in the same format as vr_lcd_get_cursor_pos().

void avr_lcd_putc(char c): writes one character to the display at the current cursor position and advances the cursor by one.

void avr_lcd_puts(const char *s): writes a NULL-terminated C string at the current cursor position using avr_lcd_putc
